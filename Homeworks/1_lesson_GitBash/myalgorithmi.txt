Решила сделать задание более интересным и выполнить его максимально без использования "мыши", т.е. через терминал и через него же запушить задание в свой GitHub. Цель - задействовать побольше шагов, чтобы лучше запомнить как работать с Git Bash.

Алгоритм действий таков:
1. Открыть терминал.
2. Посмотреть какие папки есть в директории (% ls -la).
3. Найти папку QA_course. Я знаю, что она лежит на рабочем столе, поэтому перехожу в папку Desktop (% cd Desktop)
4. Перейти в папку QA_course (% cd QA_course).
5. Создать в папке QA_course папку Homeworks (% mkdir Homeworks).
6. Создать в папке Homeworks папку 1_lesson_GitBash (% cd Homeworks) затем  (% mkdir 1_lesson_GitBash).
7. Создать в папке 1_lesson_GitBash три файла:
 README.md - описание самого ДЗ (% touch README.md);
 1_HW_terminal.txt - выполненное ДЗ (% touch HW1_terminal.txt);
 myalgorithmi.txt - описание шагов, что и как сделано (% touch myalgorithmi.txt).
8. Заполнить файл myalgorithmi.txt. Открыть (% vim myalgorithmi.txt). В самом файле нажать "i", чтобы можно было его редактировать
Данный алгоритм прописываю только сейчас, когда создан файл. Так что фактические названия папок и файлов могут немного отличаться, если я забуду сюда вернуться и исправить их.
И вот файл заполнен, жму "ect" - выйти из режима редактирования, далее  ":" + "wq" чтобы закрыть файл.
9. Заполнить файл README.md. Открыть (% vim README.md). Нажимаю "i".  Копирую задание и вставляю само задание (всеми любимые cmd + c --> cmd + v), затем "ect" - выйти из режима редактирования, далее  ":" + "wq" чтобы закрыть файл.
10. Выполняю задание в файле 1_HW_terminal.txt. Делаю в текстовом редакторе, так как тут замороченно бегать по строчкам.
11. Когда все задания выполнены "пушу" папку в GitHub.
git add*
git commit -m "Add folder for Homeworks and HW1"
git push
